Verizon operates 14,792 outdoor cabinet sites (Ericsson and Southeast markets; Samsung markets analysis pending) equipped with Eaton / Tripp-Lite precision cooling units managed by the WEBCARDLX LX Platform. These cabinets have no existing remote temperature monitoring or setpoint control capability in the standard SiteBoss alarming configuration — operators cannot view real-time cooling conditions or adjust setpoints without a physical site visit.

This initiative deploys a SiteBoss Lua script that polls each cabinet HVAC unit every 5 minutes via SNMPv2c and controls cooling setpoints remotely via the PADM20 HTTP REST API. The script is fully integrated with the existing SiteBoss infrastructure — no new hardware required.

✓ No new hardware required — leverages existing SiteBoss 550A units already deployed at cabinet sites.
✓ Dual control path — SNMP monitoring + HTTP setpoint write in a single Lua script.
✓ Fleet-ready — script accepts configurable IP, community, and credentials; deployable at scale via SiteBoss Lua slot.

2. Problem Statement

Current State — Outdoor Cabinet HVAC Blind Spots

GapCurrent StateImpact

Setpoint controlDefault factory setpoints (often 65–68°F) — no remote adjustmentOvercooling wastes energy; setpoints can only be changed on-site
Compressor/fan healthNot monitoredNo early warning for compressor failure, refrigerant loss, or VFD issues
Pressure monitoringNot monitoredHigh/low refrigerant pressure faults go unreported until unit shuts down
Cooling output (kW)Not trackedEnergy consumption unknown; no baseline for efficiency improvements
WPU overcooling5-ton WPUs cooling entire shelter for lightly loaded equipment; dead band cycling but oversized for actual heat loadWPU runs significant duty cycle to maintain 79°F room temp — far more energy than close-coupled cabinet cooling of the same equipment
⚠ Setpoint Gap: Factory default cooling setpoints on Eaton/Tripp-Lite units are typically 65–68°F. Verizon's NSTD thermal standard permits 77°F return air in most cabinet configurations. Each 1°F increase in cooling setpoint saves approximately 2–3% of HVAC energy (ASHRAE guideline). A 9°F setpoint raise represents a potential 18–27% cooling energy reduction per site.

3. Solution — SiteBoss Lua Script

The script runs inside the SiteBoss 550A Lua slot and communicates with the WEBCARDLX card over the cabinet's internal Ethernet. No VPN or WAN access to the HVAC unit is needed — the SiteBoss acts as the local proxy.

SNMP Layer — Monitoring SNMP READ ONLY

SNMPv2c GET against enterprise OID 1.3.6.1.4.1.850.1.1.3.5 (Tripp-Lite cooling). Full BER encoder/decoder in pure Lua using LuaSocket UDP — no external SNMP library required. 20 OIDs polled in a single GET request every 5 minutes:

All temperature OIDs return Integer32 in units of 0.1°F; script divides by 10 for display. MIB source: TRIPPLITE-PRODUCTS.MIB v20.0.2.

HTTP Layer — Setpoint Control HTTP READ/WRITE

Setpoint control uses the PADM20 REST API — the only remote path for cooling setpoint writes (setpoint is not exposed in the SNMP MIB).

OperationMethodEndpointNotes
AuthenticatePOST/api/oauth/tokenReturns Bearer token; cached and auto-refreshed on 401
Discover variablesGET/api/variables?filter[device_id]=1Cached after first call; run once to map setpoint variable ID
Write setpointPATCH/api/variables/{id}JSON API body; target label: "Cooling Setpoint" or "Return Air Setpoint"

4. Financial Opportunity

Basis — DT Morris Trial: Two Bard HR58BPA05EP 5-ton Wall Pack Units (WPUs), R-410A, ~5.8 kW electrical input at full load. N+1 configuration — one active unit at a time, alternating every 7 days. Active WPU cycles on dead band: setpoint 79°F → kicks on at 81°F → cools to 77°F → shuts off. Not continuous runtime — duty cycle is driven by ambient climate and internal heat load. Goal: migrate remaining equipment into Eaton 33U self-cooling cabinet; raise WPU setpoint to ~90°F or humidity-only mode; reduce WPU runtime to near zero.
